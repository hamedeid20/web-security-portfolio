# 📚 Methodologies

## Web Application Penetration Testing Methodology

### Web App Pentesting Types

* Black-Box Pentesting
* White-Box Pentesting
* Gray-Box  Pentesting

### Methodology

#### 1. Pre-Engagement

* Select Scope's ( Domains - Subdomains - API's )
* Rules of Engagement
* Timing for Test
* Rate Limitation

#### 2. Reconnaissance

- Passive Recon
    - Information Gathering without touch the Server
    - Google Dorks
    - WHOIS
    - Subdomains
    - Github Leaks

- Active Recon
    - Scanning
    - Direcotry Discovery
    - EndPoint Enumeration

#### 3. Vulnerability Analysis

- We are looking for :
    - XSS
    - SQL-Injection
    - IDOR
    - Authentication Flaws
    - Misconfigurations
    - API Issues
- The thinking stage, not the implementation stage

#### 4. Exploitation

- Exploiting the vulnerability
- Proof of Concept (POC)
- Bypass Control
- Privilege Escalation

#### 5. Reporting

- Vulnerability description
- Steps to reproduce
- Impact
- Evidence (screenshots / requests)
- CVSS score
- Fix recommendation

### Pentesting Checklist
[x] - [✔] - [ ]

#### [ ] Recon Checklist

    [ ] Identify main domain
    [ ] Subdomain enumeration
    [ ] Check robots.txt / sitemap.xml
    [ ] Find exposed files (.git, .env, backup)
    [ ] Tech stack detection (frameworks, server)
    [ ] Collect endpoints (JS files analysis)

#### [ ] Authentiction Testing

    [ ] Login brute force protection
    [ ] Account enumeration (error messages)
    [ ] Password reset flow
    [ ] Session management
    [ ] JWT validation (if exists)
    [ ] MFA bypass possibilities

#### [ ] Authorization Testing

    [ ] IDOR testing
    [ ] Horizontal privilege escalation
    [ ] Vertical privilege escalation
    [ ] API object access control

#### [ ] Input Validation

    [ ] SQL Injection
    [ ] XSS (reflected / stored / DOM)
    [ ] Command injection
    [ ] File upload vulnerabilities
    [ ] Path traversal

#### [ ] API Testing

    [ ] Broken object level authorization (BOLA)
    [ ] Missing rate limiting
    [ ] Mass assignment
    [ ] Excessive data exposure
    [ ] Improper authentication in APIs

#### [ ] File & Upload

    [ ] Upload bypass (php/js files)
    [ ] Content-type bypass
    [ ] Path traversal in upload
    [ ] File overwrite issues

#### [ ] Configuration Issues

    [ ] Default credentials
    [ ] Debug mode enabled
    [ ] CORS misconfiguration
    [ ] Exposed admin panels
    [ ] Sensitive headers leakage

#### [ ] Business Logic

    [ ] Coupon abuse
    [ ] Price manipulation
    [ ] Workflow bypass
    [ ] Race conditions

#### [ ] Session & Cookies

    [ ] Secure flag
    [ ] HttpOnly
    [ ] Session fixation
    [ ] Expiry issues

#### [ ] Reporting Checklist

    [ ] Clear title
    [ ] Severity
    [ ] Steps to reproduce
    [ ] Proof of concept
    [ ] Impact analysis
    [ ] Fix suggestion
